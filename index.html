Key Features

Everything you need, nothing you don't

A fully-featured CMS platform with a clean architecture foundation — extensible, testable, and production-ready out of the box.

Dynamic Content Types

Schema builder with 30+ field types — Text, RichText, Gallery, Relation, GeoLocation, Rating, DateRange, and more. Drag-and-drop field ordering, JSON options per field type.

Multi-Tenant

Full tenant isolation with subdomain resolution middleware. SystemAdmin cross-tenant console with impersonation, tenant provisioning, and per-tenant data partitioning.

Multilingual

Per-tenant language configuration, DB-backed admin UI i18n managed by SystemAdmin (250+ keys, tr/en), per-field localization flags, and language switcher in the content editor.

Public REST API

Per-content-type API exposure with API key + JWT auth, rate limiting (sliding window), response caching, EAV field filtering/sorting, field visibility/aliasing, and projection.

Role-Based Access

Four roles: SystemAdmin, TenantAdmin, Editor, Viewer. Per-content-type CRUD permission matrix. UI hides unauthorized buttons; action filters enforce server-side access control.

Security-First

BCrypt passwords (workfactor 12), CSRF on all mutations, path traversal guard on file operations, parameterized Dapper queries, login rate limiting, and email 2FA with SMTP.

Quick Start

Up and running in minutes

Docker spins up SQL Server automatically. All migrations are numbered SQL scripts applied with a single command.

1
Clone & configure
Clone the repository and copy docker/.env.example to docker/.env. Set a strong DB password and JWT signing key.
2
Start SQL Server
One command launches SQL Server 2022 in Docker with a named volume for data persistence.
3
Apply migrations
Run the numbered SQL scripts in order via sqlcmd inside the container. Dev seeds are idempotent — safe to re-run.
4
Run the app
Start the web project and log in with the seeded admin credentials. Your first tenant is ready.
bash — varyo-cms
# 1. Start SQL Server + apply migrations $ docker-compose -f docker/docker-compose.yml up -d $ docker exec varyo_db /opt/mssql-tools18/bin/sqlcmd \ -S localhost -U sa -P "Varyo_Dev2024!" -C \ -d Varyo -i /migrations/001_init_tenants_users.sql # (repeat for each migration file) # 2. Run the application $ dotnet run --project src/VaryoCms.Web info: Now listening on: http://localhost:5267 # 3. Open in browser and log in # Email: admin@dev.local # Pass: Admin123!
Public REST API

Headless-ready, out of the box

Any content type can be exposed as a typed REST endpoint in one click. Authentication, rate limiting, field visibility, and caching — all configurable per endpoint from the admin UI.

GET /api/v1/{tenant}/{content-type}
Tenant resolved from URL slug — fully isolated
Auth: None / API Key (X-API-Key) / JWT Bearer
EAV field filtering via filter[slug]=value
Sorting with sort=field:asc|desc
Field projection: ?fields=title,slug,image
Sliding-window rate limiting with X-RateLimit-* headers
Configurable response caching (per content type)
Field aliasing: rename keys in JSON response
Relation fields auto-expand to {id, displayValue}
Media fields auto-expand to {id, url}
